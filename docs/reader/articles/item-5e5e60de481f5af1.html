七國政府開源軟體公開活動國際比較報告

報告分析日本、愛沙尼亞、新加坡、德國、法國、英國及美國政府機關在 GitHub 公開的 OSS。英美的儲存庫與拉取請求規模明顯領先，法國居中，其餘四國規模較小；日本共有626個儲存庫、約3,800筆拉取請求，並以地圖、都市及空間資訊專案較具特色。報告也觀察到部分國家的公開量成長與 OSS 政策或數位政府機構成立時間重疊，但尚不足以證明因果關係。研究只涵蓋 GitHub，未納入其他程式碼平台及政府內部協作，因此適合作為比較起點，而非各國行政 OSS 能力的完整排名。

全球 OSS 趨勢報告：政府機關開源軟體（OSS）公開活動國際比較調查報告書

公開日期：2026年3月4日

最後更新日期：2026年3月25日

執筆：福地 弘行（IPA）

岡本 晉（IPA 專門委員、一般社團法人 monlon）

執行摘要（Executive Summary）

調查背景與目的

近年來，開源軟體（OSS, Open Source Software）不只在民間企業，在政府機關中的重要性也與日俱增。在德國與愛沙尼亞，依法強制規定以公帑開發的軟體必須開源；而在美國，則將公開 OSS 作為聯邦政府降低成本的對策，進而推動行政系統的重複利用。各國政府採取的方法相當多元。在學術研究方面，雖然著眼於政策框架與組織的質性調查已有所進展，但針對實際 OSS 公開活動數量與其規模的國際比較仍顯不足，特別是日本在國際上的定位依然不明朗。因此，本報告基於「政府機關 OSS 的公開數量如何因國家或地區而異？」的研究問題（Research Question），利用 GitHub 上的數據進行了定量分析（Quantitative Analysis）。本調查針對包含日本在內的 7 個國家（日本、愛沙尼亞、新加坡、德國、法國、英國、美國）的政府及公營機構在 GitHub 上公開的 OSS，使用儲存庫（Repository）數量、拉取請求（Pull Request）數量、星數（Star）等多個指標進行定量分析，藉此釐清各國 OSS 活動的特徵以及與政策之間的關聯性。

主要調查結果

第一，調查結果顯示各國的 OSS 活動存在明顯的規模差異。

英國（儲存庫約 16,000 件、拉取請求約 190 萬件）與美國（儲存庫約 10,000 件、拉取請求約 54 萬件）屬於大規模；法國（儲存庫約 2,400 件、拉取請求約 24 萬件）屬於中規模；日本、愛沙尼亞、新加坡、德國則屬於小規模。從數據中呈現的日本特徵來看，例如在 626 件儲存庫中，拉取請求約有 3,800 件，平均每件儲存庫的比例約為 6.1 件，在小規模組別中是最低的。這顯示透過拉取請求進行開發活動的頻率相對較低，反映出其重視嚴格審查流程與品質管理的特徵。

第二，觀察發現各國 OSS 公開數量的增加時期與 OSS 相關政策的導入時期，在時間點上有一定的重疊。

例如英國成立政府數位服務局（Government Digital Service, GDS, 2012 年）、美國推行聯邦原始碼政策（Federal Source Code Policy, 2016 年）、法國實施艾羅通令（Circulaire Ayrault, 2012 年），以及德國提出主權技術基金（Sovereign Tech Fund, 2021 年）構想等，各國在政策施行前後皆可見 OSS 公開數量的增長。日本也在 2012 年的電子政府開放資料策略、2021 年數位廳（Digital Agency）成立的前後，出現公開數量增加的情形。

第三，在技術領域的分布上，觀察到了全球共通的趨勢以及各國特有的特色領域。

在 7 個國家的整體數據中，行政服務（2,340 件）與數據基礎設施及 API（1,935 件）的公開數量最多；同時也觀察到各國特有的重點領域，例如愛沙尼亞以 X-Road 為首的資料交換平台（Data Exchange Layer），以及新加坡的智慧城市相關領域。至於日本，其特色在於國土地理院與國土交通省在地圖與都市資訊領域的 OSS 公開活動。

對日本的啟示

雖然日本政府機關的 OSS 活動與英國、美國、法國相比規模較小，但已確認國土交通省與國土地理院在地圖及空間資訊領域有相當活躍的表現。整體而言，自數位廳成立以來，公開數量也呈現增加趨勢。未來為了加速這些萌芽中的發展，並將其擴大為更具組織性的行動，需要採取策略性的措施。

未來展望

本調查僅限於 GitHub 上的定量分析方法，其他平台或利用政府內部開源（InnerSource）進行的 OSS 公開活動並未納入對象範圍。此外，雖然掌握了各國 OSS 活動實際狀況的整體輪廓，但其背後的政策或政治動向與活動之間的因果關係仍未被釐清。今後除了必須持續進行突破上述限制的研究外，也有必要加速討論促使政府機關推廣 OSS 活動的具體方案。
