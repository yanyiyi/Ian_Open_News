100天後：德國政府的自由軟體推動力仍不足

新聞

100天後，德國政府仍缺乏自由軟體的推動

2025年8月14日，德國聯邦政府執政滿一百天。基督教民主聯盟（CDU）、社會主義者（SPD）和基督教社會聯盟（CSU）在選前承諾了雄心勃勃的自由軟體目標，但這些目標具體的內容、實現方式以及自由軟體對保障德國技術主權的角色仍不清晰。

2025年2月，歐洲自由軟體基金會（FSFE）呼籲新上任的政府將公共數位基礎設施的可控性、安全性與主權列為首要任務，並透過持續採用自由軟體（即開放原始碼）來實現。我們的核心訴求包括：「公共資金？公共程式碼！」、確保自由軟體的長期資助、提升能力建構，以及統計監測進展情況。將此願景落實為現實需要真正的政治推動力：所有用於軟體的公共資金都必須僅限於自由軟體！

執政滿一百天，是時候進行檢視。

採購改革：自由軟體應被優先考量

距今幾日，即2025年8月6日，聯邦內閣通過了一項草案法律，旨在加速公共採購程序，使其更高效、靈活且數位化。立法內容雖包含在招標標準中考慮數位主權的可能性，但未能把握機會將自由軟體優先納入採購考量。現在，聯邦議院的執政黨團需要加強該草案內容。

FSFE呼籲「公共資金？公共程式碼！」原則應明確嵌入公共採購， 給予自由軟體明確的優先權。這是終結對專有軟體及其供應商的結構性依賴、實現真正技術主權的唯一途徑。詳細訴求。公部門還應被強制要求定期測量並公開其採購、委託或開發的軟體中自由軟體的比例。這將使進展可衡量，有助於發現不良發展。

「德國棧」：全國獨力行動還是歐洲合作？

根據數位部長 Karsten Wildberger 的說法，到 2028 年，政府計劃推出「德國棧」，一個統一的 IT 架構，其中包括雲端軟體等基礎元件以及明確定義的介面。然而，僅專注於純國家的 IT 棧是行不通的路。相反地，歐洲各成員國應成為夥伴，共同合作採購與開發，重點在於互操作性與共享解決方案，而無需不斷重複創造新架構。若無 EU 層級的協調與合作，「德國棧」恐將淪為孤立項目。

目前也不清楚這個「德國棧」是否真會以自由軟體（Free Software）的形式發展。透明度令人質疑：在七月早先非公開的過渡性簡報中，民權組織、自由軟體專案及其社群的代表被排除在外。

自由軟體倡議者 (FSFE) 的呼籲

自由軟體倡議者（FSFE）呼籲政府在發展 IT 架構時採取歐洲層級的思維與行動，與歐洲夥伴共同合作建立一個聯合架構，而非將資源投入到可能失敗的國家獨力項目。無論如何，聯邦數位化暨政府現代化部 (BMDS) 都必須承諾採用自由軟體及相容於自由軟體的開放標準，以創造真正的替代方案，對抗專有產品。從一開始就應讓民權團體與自由軟體產業參與，利用現有專業知識、促進接受度並推動向技術主權邁進的真正變革——在聯邦、州及市級各層面，以及整個歐洲。

聯邦預算：缺乏對自由軟體的穩定長期資金支持

在發表 2025 年聯邦預算與國會預算辯論之後，政府如何確保對自由軟體提供穩定且長期的資金支持仍不清楚。目前看不到將奢華支出於專有授權的「公共金錢？公共程式碼！」原則下，重新分配到自由軟體的跡象。包括聯邦內政部的自由軟體預算也過少，處境依然 precarious（危險）。

FSFE 的要求：政府提供安全、永久的公共領域自由軟體資金。 除了採購轉型，這還需要針對中央數位基礎設施中的自由軟體提供有目標性的財務支持。這必須在這一份以及所有未來的聯邦預算中明確反映出來。此外，尚待準備的 BMDS 預算也必須包括對數位主權中心 (ZenDiS) 的可靠資金支持，使其能夠透過自由軟體支持公共機構邁向技術主權之路。

“如果德國政府真要實現數位主權，就必須承諾採用自由軟體。只有自由軟體才能確保供應商的獨立性、互通性和創新自由，”說 德國聯邦自由軟體聯盟 (FSFE) 的資深政策專案經理 Johannes Näder 表示。 “這需要為自由軟體和聯邦政府自己的自由軟體計畫（特別是 ZenDiS）提供安全且長期的資金支持。自由軟體必須在公共採購中獲得優先考量，落實『公帑？公碼！』原則。任何 IT 堆疊若非完全以自由軟體開發——而是封閉於國內 silo 中——將難以成功。這需要與歐盟、其他成員國以及自由軟體生態系緊密合作。”

背景：政府自己的承諾

新成立的數位部旨在作為數位轉型的推動者，以及一個現代化且具能力的行政機構。自由軟體被期望扮演重要角色。在他們的執政聯盟協議中，CDU、CSU 和 SPD 承諾：

“我們將在各個層級定義開放介面和開放標準，並策略性地促進歐洲生態系中的開源活動。 […] 為此，我們將策略性地調整 IT 預算，並為開放源碼設定雄心勃勃的目標。”

2025 年 5 月，在 re:publica 大會上，數位部長 Wildberger 重申了這一目標，表示自由軟體和開放標準應該成為聯邦政府數位基礎設施的「指導原則」。在此背景下，政府計劃在 BMDS 下建立一個新的數位機構，將 ZenDiS 和 DigitalService 結合起來，並可能整合其他目前獨立的機構。

在 re:publica 的演說中，部長與批判性社群的互動可被視為一個歡迎的第一步。然而，這一步必須由公民社會的真誠、制度化的參與所追隨。這對於推動公共行政的數位轉型至關重要，這種轉型將使每個人受益，並使德國與歐洲在經濟上強大且足以應對未來挑戰。行動的時機已經到來。

自由軟體與「公帑？公共程式碼！」

自由軟體賦予每個人使用、研究、分享和改進應用程式的權利，以達成任何目的。這四項自由確保類似的應用程式無需重新編寫，並透過透明的流程，避免他人重複發明輪子。在大型專案中，專業知識與成本可以被共享，應用程式對所有人開放。這促進了創新，並節省了未來的納稅人支出，同時將對廠商的依賴降至最低，使安全問題更容易修復。自由軟體基金會歐洲（Free Software Foundation Europe）因此呼籲「公帑？公共程式碼！」——如果這是公帑，那麼它也應是公共程式碼！更多資訊可於 Public Code 網站。
