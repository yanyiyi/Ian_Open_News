Claude Tag 哪裡不一樣？Anthropic 的「數位同事」與導入前 4 個提醒

  • AI 人工智慧
  • 廖紹伶

Anthropic 推出名為 Claude Tag 的新產品，把搭載最新模型 Opus 4.8 的 AI 直接嵌進 Slack，成為整個團隊都能透過輸入 @Claude 來指派工作的常駐成員。表面上，這像是一次 Slack 整合工具的改版，取代 Anthropic 原有的 Claude in Slack 應用，但特斯拉前 AI 總監、OpenAI 創始成員、現任職於 Anthropic 的 Andrej Karpathy 給了它一個更大的定位：這是大型語言模型（LLM）介面的第三次重大轉變。

根據 Karpathy 的說法，第一個典範，是 LLM 作為一個你前往的網站；第二個典範，是它變成一個下載到電腦裡的 App；而第三個，則是它成為一個自我完整、持續存在、可非同步運作的實體，擁有組織級的工具與脈絡，與人類團隊並肩工作。他形容，一旦把工具、整合、運算環境、記憶、安全等底層工程都做到「就是能用」，Claude 基本上就以一種無縫的方式加入了團隊，你可以像跟人說話一樣跟它溝通。

對企業來說，當一個 AI 要在你的溝通核心拿到常駐席位，你的採購談判與治理框架，準備好了嗎？這篇會先說明 Claude Tag 怎麼運作，再回到這個決策問題。

不是單人工具，而是全頻道共用的「數位同事」

《VentureBeat》指出，真正讓 Claude Tag 與過往整合工具拉開差距的，是四項能力。

第一，它是多人共用的：在一個 Slack 頻道裡，只有一個 Claude 與所有人互動，而不是每位使用者各自擁有一個分身，任何人都能看到它正在做什麼，也能從上一個人停下的地方接手。這與多數現有的 Slack AI 整合形成對比，後者多半是單人工具。

第二，它會隨時間學習。《TechCrunch》報導，Claude 跟著頻道進展，持續累積關於該頻道工作的脈絡，使用者不必每次都從頭解釋專案；若獲得授權，它也能自動從組織其他頻道蒐集資訊。《VentureBeat》補充，Anthropic 表示它不會從私人頻道回報內容。

第三，它具備主動性。在開啟 ambient（環境感知）模式後，Claude 會主動加入對話，浮現組織各處的相關資訊，並追蹤那些被遺忘的討論串或任務。《VentureBeat》形容，這是一種代理能力的顯著擴張：Claude 不只是回應請求，而是監看資訊環境，並自行判斷人類隊友需要知道什麼。

第四，它能非同步工作，在數小時到數天的跨度裡自主推進專案。Anthropic 表示自家團隊「現在花更多時間把任務平行地委派給許多個 Claude」。

值得關注的是，Anthropic 自家產品團隊目前有 65% 的程式碼，是由內部版本的 Claude Tag 所產出，該公司也透過同一套系統運作內部支援與資料洞察頻道。換句話說，Anthropic 主張，它自己產品工程的多數產出，已經流經 Claude Tag。

企業 AI 最競爭的一塊地，在 Slack 頻道裡

Claude Tag 登場的時機，正值企業 AI 中競爭最激烈的一塊領地成形：Slack 頻道。

Slack 本身一直積極把平台定位為「代理式作業系統」，而主要的 AI 業者也以競相插旗作為回應。Salesforce 在 2021 年以 277 億美元收購 Slack，今年 3 月為 Slackbot 發表超過 30 項新功能，是收購以來最大幅度的翻修，把它從單純的對話助手轉變為全方位的企業代理。

OpenAI 則在 4 月推出「Workspace Agents」，讓企業訂戶能設計可跨第三方應用執行工作任務的代理，涵蓋 Slack、Google Drive、Microsoft 應用、Salesforce 與 Notion。Perplexity 推出企業版「Computer」代理並直接整合 Slack，讓員工能在頻道裡直接呼叫 @computer。Cognition 的自主軟體工程師 Devin 從早期就以 Slack 作為主要介面。連 Microsoft 都把 GitHub Copilot 帶進了 Teams。

《VentureBeat》分析這股匯流背後的邏輯：一般企業要同時應付超過 1,000 個應用，員工耗費大量時間在情境切換上，生產力因此被拖累達四成。因此，誰成為工作協調所在的溝通層裡的預設存在，誰就取得龐大的分發以及資料優勢。常駐在工作實際發生頻道裡的 AI，會吸收讓它越來越難被替換的組織脈絡。

值得一提的是，把脈絡視為兵家必爭之地的不只 Anthropic。《TechCrunch》指出，Microsoft 有透過 Copilot 與 Work IQ 表現的 Graph；Snowflake 與 Databricks 把自家平台定位為承載組織隱性知識、供代理取用的後端；Glean 也在打造一個理解公司脈絡、介於模型與企業資料之間的智慧層。

支撐 Anthropic 這項押注的財務數字相當可觀。據《VentureBeat》，公司在 5 月底完成 650 億美元的 H 輪募資，投後估值達 9,650 億美元，本月稍早年化營收跨過 470 億美元；其中 Claude Code 的年化營收已成長至超過 25 億美元，自 2026 年初以來增加一倍有餘，企業用途已占 Claude Code 營收的一半以上。

這些數字解釋了 Anthropic 為何在頻道層級的存在感上投入如此之重：每一個授予 Claude 持續存取頻道權限的企業客戶，都代表比一次聊天對話或一次 API 呼叫更深的整合，使用黏著度更高，token 消耗更大，轉換成本也隨之升高。

讓 Claude Tag 常駐前，企業要先想清楚的幾件事

正因為 Claude Tag 要的是頻道裡的一個常駐席位，《VentureBeat》指出，Anthropic 把企業級隔離放在系統設計的核心。系統管理員為不同用途定義各自獨立的 Claude 身分，每個身分都被限定在特定頻道、搭配特定工具與資料存取權，包括 Claude 累積的記憶在內，一切都留在這些邊界之內。《TechCrunch》舉例，一個為法務工作設定的 Claude，不會把記憶滲入工程頻道。

據《VentureBeat》，管理員能在組織與頻道兩個層級設定 token 花費上限，並能檢視一份完整日誌，記錄 Claude 採取過的每一個動作、以及每項任務是由哪位使用者所要求。對需要管理合規、稽核或法規要求的組織而言，這套日誌與權限範圍的架構是基本門檻，而過去一年，正是這類機制的缺席，讓不少企業在評估 AI 協作工具時卻步。

即便如此，《VentureBeat》也點出買方仍有幾件事需要審慎權衡。第一是供應商綁定：當一個 Claude 已累積數月的頻道脈絡與組織記憶，它會變得非常難以替換，習慣以多雲彈性進行採購談判的團隊，得認真想清楚，把單一供應商的 AI 永久接進組織知識所在的溝通層意味著什麼。

第二是長駐監看的治理：主動模式下，Claude 會監看頻道並浮現它判定為相關的資訊，組織需要為「一個不只回應請求、還主動巡查資訊流並對人類該知道什麼做出編輯判斷的 AI」建立清楚的框架，對受監管產業而言，這牽涉現有 AI 治理政策可能尚未涵蓋的問題。

第三是計價：Anthropic 尚未公布 Claude Tag 的詳細定價，只說明它採 token 用量計費並附帶管理控制，而對一個持續監看頻道、建立記憶、跨數小時到數天非同步工作的代理來說，token 消耗的樣態可能與傳統 AI 用量很不一樣。

第四是可靠度：Anthropic 近月對需求激增造成的基礎設施壓力一向坦率，而對一個被定位為全天候團隊成員的產品，當機所付出的代價，與一個隨叫隨用的工具並不相同。

從「儲存資料的系統」到「坐在現場的代理」

Claude Tag 並非憑空出現。Anthropic 在 2025 年 10 月首次將 Claude 與 Slack 整合，當時聚焦個人生產力；2026 年 1 月推出包含 Slack、Canva、Figma 等的互動式 Claude apps；同年 4 月推出可在規模上建構與部署雲端代理的 Claude Managed Agents；接著是 5 月底的 Opus 4.8，Anthropic 形容它「判斷更敏銳、對自身進度更誠實，並能比前代獨立工作更久」。Claude Tag 正是把前述能力綜合於一身的產物，Anthropic 把它定位為「Claude Code 演進的起點」。

而這次推出更深層的意義，《VentureBeat》認為，或許在於它揭示了重塑企業軟體的競爭動態。數十年來，商業科技中最有價值的位置一直是記錄系統：資料庫、CRM、ERP。當前的 AI 軍備競賽則指向另一個方向：下一個時代的企業價值，可能不再由儲存資料的系統掌握，而是由那個坐在工作現場、知道該拿這些資料怎麼辦的代理所捕捉。對企業科技領導者而言，問題已不再是這樣的代理會不會到來，而是當它到來時，自己是否已準備好管理它。
