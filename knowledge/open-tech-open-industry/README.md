# 開放科技與開放產業發展

這條主線收錄開源、開放資料、資料治理、授權、標準、公共數位基礎建設、AI governance 與開放產業案例。

## 工作區

- `research/`：一手文件、標準、政策、技術文件、GitHub repo 對照。
- `briefs/`：可 review 的知識 brief。
- `published/`：已定稿或已發布的整理。

## 收錄判斷

一筆資料值得進一步處理，通常至少符合其中一項：

- 涉及開放資料、開源、開放標準、開放授權或公共數位基礎建設。
- 說明政策、產業、社群或技術趨勢。
- 能追蹤後續法規、採購、標準、資安、供應鏈或國際組織動向。
- 對台灣、OCF、公共利益或開放生態系有參考價值。

## 審查底線

- 技術、法規、標準、授權名稱必須可查證。
- 不把廠商新聞稿或 marketing language 當成已證實事實。
- 明確區分新聞事實、推論、風險與建議。
- 產業機會必須附限制與來源，不只列出樂觀敘事。
